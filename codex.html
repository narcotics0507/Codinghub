导读

这份教程适合第一次配置 Codex 的同事。真实落地时，把示例地址、模型名和截图替换成你的团队资料。

建议先确认账号、API 密钥和模型服务地址，再开始改本机配置。不要把真实密钥写进网页、截图或公开仓库。

1. 准备信息

信息用途
Base URL告诉客户端请求哪个模型服务入口。
API Key个人密钥，用于鉴权和额度统计。
模型名决定默认使用的文本模型。

2. 写入配置

先创建配置目录，再把团队提供的配置写入本机文件。

shell
mkdir -p ~/.codex
toml
model_provider = "OpenAI"
model = "your-model-name"

[model_providers.OpenAI]
name = "OpenAI"
base_url = "https://example.com/v1"
env_key = "OPENAI_API_KEY"

常见问题

密钥能不能共用？

不建议。每个人使用自己的密钥，方便额度统计和问题排查。

配置后没有生效？

检查配置文件路径、模型名、环境变量和客户端是否重启。
