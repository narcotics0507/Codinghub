导读

本地 Agent 类工具通常涉及面板、服务进程、模型服务商和消息渠道。教程要先讲清角色，再讲操作。

1. 准备角色

角色说明
管理面板负责安装、检测和配置。
Agent 服务真正处理消息和任务。
模型服务商提供文本或图片模型能力。
消息渠道把用户消息送到 Agent。

2. 配置模型

新增服务商后填写 Base URL、API Key 和模型名。保存后先做一次最小测试。

测试问题
请回复“模型服务已连通”，并用一句话说明当前时间。

3. 接入消息渠道

1
创建渠道配置

复制平台提供的回调地址和密钥。

2
保存凭据

不要把 token 放在公开文档或仓库里。

3
发送测试消息

确认消息能进入 Agent 并返回。

4. 检测修复

遇到问题先确认服务进程、端口占用、模型服务响应、渠道签名和网络连接。把错误截图补到本节会更利于团队排查。
