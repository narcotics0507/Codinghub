Cloudflare Pages

把静态教程站部署到 Cloudflare Pages

这个仓库不需要构建命令。连接 GitHub 后，Cloudflare Pages 可以直接把根目录作为输出发布。

部署目标：GitHub 仓库 + Cloudflare 托管域名。

教程目录

准备 连接 Git 构建配置 绑定域名

准备

需要 Cloudflare 账号、已经托管在 Cloudflare 的域名，以及一个包含本站文件的 GitHub 仓库。

1. 连接 Git 仓库

1
进入 Workers & Pages

选择创建 Pages 项目。

2
连接 GitHub

授权后选择 `narcotics0507/Codinghub` 仓库。

3
选择 main 分支

后续 push 到 main 会触发自动部署。

2. 构建配置

字段值
Framework presetNone
Build command留空
Build output directory/，如果页面不接受就填 .
Root directory留空
本地预览
python3 -m http.server 4173

3. 绑定域名

部署成功后先访问 `*.pages.dev` 地址。确认没问题后进入 Custom domains，添加你的子域名。域名已在 Cloudflare 托管时，DNS 记录通常会自动创建。

建议先用子域名，例如 `docs.your-domain.com`，避免影响主站。
